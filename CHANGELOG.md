# Changelog · finance-bi-dashboard

## [3.0.2] - 2026-07-16 · 脱敏发布（公开分发版）
**性质**：非功能性变更，与 v3.0.1 分析逻辑、勾稽校验、产物完全一致；仅做去标识化以便公开上传 GitHub。

### 脱敏替换明细（仅占位符，不影响算法）
> 出于公开分发要求，原仓库中的真实主体名 / 往来单位名 / 区域码 / 本地绝对路径**已全部移除**，统一替换为下列通用占位符。此处不列示原始值，避免再次暴露。
| 类别 | 替换为（通用占位符） |
|---|---|
| 真实公司全称 / 简称 / 看板标题 | `示例科技有限公司` / `示例科技` / `示例科技 · 经营分析控制台` |
| 真实往来客户（甲方 / 乙方） | `示例客户甲` / `示例客户乙` |
| 真实关联方 | `示例关联方` |
| 真实区域码 | `示例区域` |
| 本地绝对路径（资料库 / 看板输出） | 相对目录 `data` / `output` |
| 本地 venv / 技能安装绝对路径 | 自动探测（`__file__` / 环境变量 `FINANCE_BI_PY` / 回退 `sys.executable`） |

### 刻意保留的非敏感项（非 PII，仅为文件/表名）
- 源文件名 `2026-6-30AR.xlsx`、`2026开票记录.xlsx` 等业务资料文件名——属用户资料库命名约定，非公司标识。
- 工作表名 `应收款明细`——属 Excel 内部表名，非敏感信息。
- 目录名 `经营看板`——通用中文业务词，非专属标识。

### 校验（发布前必过）
- 敏感字面量扫描（公司名 / 往来单位名 / 区域码 / 本地绝对路径 / 本地用户名）→ **0 命中**。
- 校验台：`validate_skill.py --loops 1` PASS 1/1；`--loops 10` PASS 10/10；`--loops 100 --skip-extract` 待回归。
- 全部脚本 `py_compile` 通过；`validate_dash.js` `node --check` 通过。
- 真实交付物（`data.json` / `dashboard.html` / `*.pptx` / `*.pdf` / `*.docx` / 审计底稿 xlsx）均已被 `.gitignore` 排除，绝不入库。
